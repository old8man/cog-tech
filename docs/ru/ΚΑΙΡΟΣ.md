## Преамбула

ΚΑΙΡΟΣ (от греч. καιρός — "решающий момент") — это архитектурно-агностический формальный инструмент для категориальной верификации когнитивных систем. В отличие от v1.0, которая неявно опиралась на конкретные реализации (VIT, φ, метрика Фишера), версия 2.0 формулирует **необходимые условия** в терминах **предельных конструкций** (colimits, functors, natural transformations), применимых к любой архитектуре.

**Архитектурное различие:** Augmentation-подходы (prompt engineering, RAG, tool use) оперируют на фиксированной модели M, не изменяя её внутреннюю динамику. Формально: augmentation ∈ End(Output), но augmentation ∉ End(Dynamics).

Это означает, что augmentation-системы не удовлетворяют условию **Ω6.1 (Рефлексивный оператор)**: отсутствует φ: S → S, где φ(s) модифицирует внутреннее состояние на основе самонаблюдения.

По **Теореме о минимальном порядке** (§Преамбула), система порядка < 3 не может устойчиво поддерживать себя в изменяющейся среде. Augmentation-системы имеют порядок 1 (реактивность на среду без рефлексии динамики).

Это не оценочное суждение об "истинности" интеллекта — это архитектурная констатация: augmentation и closure решают разные задачи с разными гарантиями.

### Теоретический фундамент: Кибернетика порядка N

Кибернетика разных порядков определяет уровни рефлексивности системы:

| Порядок | Объект наблюдения | Характеристика |
|---------|-------------------|----------------|
| **1-й** | Среда (E) | Система наблюдает и реагирует на среду |
| **2-й** | Наблюдение среды | Система наблюдает своё наблюдение (рефлексия) |
| **3-й** | Процесс рефлексии | Система моделирует собственную рефлексию |
| **N-й** | Рефлексия^{N-1} | Вложенная мета-рефлексия порядка N |

**Определение (Когнитивная система порядка N):**
```
Система A имеет порядок N ⟺
  ∃ оператор φ: S → S такой что φ^N(s) ≉ φ^{N-1}(s), но φ^{N+1}(s) ≈ φ^N(s)
```

**Теорема (Минимальный порядок):** Когнитивная система, способная к устойчивому самоподдержанию в изменяющейся среде, имеет порядок ≥ 3.

*Доказательство (sketch):*
- Порядок 1: реактивная система, не может предвидеть изменения
- Порядок 2: может моделировать себя, но не может корректировать модель
- Порядок 3: может корректировать свою самомодель → устойчивость

### Категориальная основа

ΚΑΙΡΟΣ 2.0 формулирует условия в терминах теории категорий:

- **Объекты:** состояния, процессы, агенты
- **Морфизмы:** преобразования, динамика, взаимодействия
- **Функторы:** отображения между категориями (восприятие, действие)
- **Естественные преобразования:** согласованные преобразования функторов
- **Пределы/копределы:** универсальные конструкции (продукты, копроизведения)

### Принципы ΚΑΙΡΟΣ 2.0

1. **Архитектурная агностичность:** Условия формулируются без предположений о конкретной реализации
2. **Категориальная полнота:** Каждая категория представляет независимый аспект когниции
3. **Предельная формулировка:** Конкретные реализации — как экземпляры универсальных конструкций
4. **Фальсифицируемость:** Каждое условие имеет критерий провала
5. **Масштабная инвариантность:** Условия сохраняются при композиции систем

## Ω1. ОНТОЛОГИЧЕСКОЕ ОСНОВАНИЕ

*Категория: Что система ЕСТЬ*

### Ω1.1 Единство порождающего принципа

> Существует ли единый порождающий принцип, из которого выводимы все остальные структуры?

**Универсальное требование:**
```
∃! Axiom: ∀ Structure S ∈ Architecture: S ⊢ Axiom
```
(Существует единственная аксиома, из которой выводима каждая структура)

**Категориальная формулировка:**
```
Архитектура образует категорию C с начальным объектом ⊥ (аксиома)
∀ объект X ∈ C: ∃! морфизм ⊥ → X (вывод)
```

**Реализации:**

| Архитектура | Аксиома |
|-------------|---------|
| SYNARC | Самоподдержание существования |
| FEP | Минимизация свободной энергии |
| GWT | Глобальное рабочее пространство |
| ACT-R | Рациональный анализ |
| IIT | Интегрированная информация |

**Критерий выполнения:** Явная демонстрация вывода для каждой структуры
**Критерий провала:** Наличие структур, требующих дополнительных аксиом

### Ω1.2 Операционное замыкание

> Определяется ли система через собственные операции, а не внешние критерии?

**Универсальное требование:**
```
∃ оператор Φ: S → Measures такой что:
(a) Φ определён через внутренние операции системы
(b) Область существования V = {s : Φ(s) удовлетворяет критерию C}
(c) C также определяется эндогенно (не внешним оракулом)
```

**Категориальная формулировка:**
```
Система образует алгебру над собой: End(S) ⊂ Hom(S, S)
Операционное замыкание: ∀ внутренний процесс f ∈ End(S)
  f(s) определяется через элементы S, не через внешние E
```

**Реализации:**

| Архитектура | Φ (функция существования) | V (область) |
|-------------|---------------------------|-------------|
| SYNARC | VIT (Viability Interval Time) | {s : VIT(s) > θ} |
| FEP | Variational Free Energy | {s : F(s) < F_max} |
| Autopoiesis | Организационная замкнутость | {s : организация сохраняется} |
| Enaction | Sense-making | {s : смыслопорождение возможно} |

**Критерий выполнения:** V определяется через Φ, а Φ — через внутренние операции
**Критерий провала:** V задаётся внешним оракулом или не определена формально

### Ω1.3 Экзистенциальный разрыв

> Содержит ли система необходимый разрыв между моделью и реальностью?

**Универсальное требование:**
```
Φ = Φ_endo + Φ_gap
где Φ_gap(s, e) > 0 при e ≠ pred(s, e)

Полное замыкание (Φ_gap ≡ 0) невозможно для сознающей системы
```

**Категориальная формулировка:**
```
Функтор восприятия O: Env → State не является изоморфизмом
Существует естественное преобразование gap: O·P → Id с ker(gap) ≠ ∅
где P: State → Env — функтор действия
```

**Критерий выполнения:** Экзистенциальный разрыв эксплицитно признаётся и используется
**Критерий провала:** Солипсистское замыкание или наивный реализм

### Ω1.4 Конструктивность

> Являются ли все определения вычислимыми?

**Универсальное требование:**
```
∀ Definition D: ∃ Algorithm A: A(input) = D(input) за конечное время
```

**Критерий выполнения:** Каждое определение сопровождается алгоритмом вычисления
**Критерий провала:** Неконструктивные (чисто экзистенциальные) определения

### Ω1.5 Минимальность

> Является ли аксиоматический базис минимальным?

**Универсальное требование:**
```
∀ axiom a ∈ Axioms: ¬(Axioms \ {a} ⊢ a)
```

**Критерий выполнения:** Доказательство независимости каждой аксиомы
**Критерий провала:** Избыточные или зависимые аксиомы

## Ω2. ГРАНИЦА И ЗАМЫКАНИЕ

*Категория: Самость и не-самость*

### Ω2.1 Определённость границы

> Существует ли формальная граница между системой и средой?

**Универсальное требование:**
```
∃ разбиение: Universe = System ⊔ Environment
Граница ∂System определена и динамична
```

**Категориальная формулировка:**
```
Расслоение π: Total → Base с сечениями σ: Base → Total
System = Im(σ), Environment = Total \ Im(σ)
```

**Реализации:**

| Архитектура | Механизм границы |
|-------------|------------------|
| SYNARC | Markov blanket в графовой модели |
| FEP | Статистическая независимость через blanket |
| Autopoiesis | Самопорождающая мембрана |
| Enaction | Структурное сопряжение |

**Критерий выполнения:** Граница определена формально и эндогенно
**Критерий провала:** Граница задана извне или не определена

### Ω2.2 Топология пространства состояний

> Определено ли пространство состояний S как математический объект?

**Универсальное требование:**
```
S является:
(a) Множеством с определённой мощностью
(b) Топологическим пространством с σ-алгеброй
(c) Имеет метрическую/дифференциальную структуру (если применимо)
```

**Категориальная формулировка:**
```
S — объект в категории Top (топологические пространства)
или Man (многообразия), или Meas (измеримые пространства)
```

**Критерий выполнения:** Явное определение (S, τ) или (S, τ, g) с указанием категории
**Критерий провала:** S как "пространство признаков" без формализации

### Ω2.3 Компактность и ограниченность

> Ограничены ли состояния системы?

**Универсальное требование:**
```
S компактно или ∃ R_max: ∀s ∈ S: ‖s‖ ≤ R_max
Ресурсы конечны: dim(S) = D < ∞ (для конечных систем)
```

**Критерий выполнения:** Явные ограничения с физическим/когнитивным обоснованием
**Критерий провала:** Неограниченность без обоснования

### Ω2.4 Область существования

> Определена ли область жизнеспособности V ⊂ S?

**Универсальное требование:**
```
V = {s ∈ S : система продолжает существовать в s}
∂V — граница существования
```

**Категориальная формулировка:**
```
V — подобъект S с каноническим вложением i: V ↪ S
∂V — topological boundary в индуцированной топологии
```

**Реализации:**

| Архитектура | Определение V |
|-------------|---------------|
| SYNARC | {s : VIT(s) > θ} или {s : ‖σ_sys(s)‖_∞ < 1} |
| FEP | {s : F(s) < F_max} |
| Homeostasis | {s : все гомеостатические переменные в норме} |
| Autopoiesis | {s : организация сохраняется} |

**Критерий выполнения:** V определена формально, ∂V характеризована
**Критерий провала:** V не определена или задана перечислением

## Ω3. ПЕРСИСТЕНТНОСТЬ

*Категория: Сохранение во времени*

### Ω3.1 Уравнение динамики

> Определена ли эволюция состояния во времени?

**Универсальное требование:**
```
∃ функция эволюции F: S × E × T → TS (или эквивалент)
ds/dt = F(s, e) или s_{t+1} = F(s_t, e_t)
```

**Категориальная формулировка:**
```
Динамика — функтор Time → Endo(S)
t ↦ (s ↦ s(t)) определяет поток на S
```

**Реализации:**

| Архитектура | Форма динамики |
|-------------|----------------|
| SYNARC | ds/dt = −β·∇_Γ VIT + F_ext + F_int |
| FEP | ds/dt = −∇F + Q·∇·∇F + noise |
| Neural ODE | ds/dt = f_θ(s, t) |
| Discrete | s_{t+1} = f(s_t, e_t, a_t) |

**Критерий выполнения:** Полное уравнение со всеми компонентами
**Критерий провала:** "Система обновляет состояние" без формулы

### Ω3.2 Декомпозиция динамики

> Разложима ли F на интерпретируемые компоненты?

**Универсальное требование:**
```
F = ⊕_i F_i с указанием семантики каждого F_i
Компоненты: внутренняя, внешняя, активная, мнемоническая, ...
```

**Критерий выполнения:** Каждый компонент имеет независимую интерпретацию
**Критерий провала:** F как "чёрный ящик"

### Ω3.3 Функция стабильности

> Существует ли функция, гарантирующая сохранение системы?

**Универсальное требование:**
```
∃ L: S → ℝ (функция Ляпунова или аналог):
(a) L(s) характеризует "удалённость от разрушения"
(b) L̇(s) < 0 вне равновесия (диссипативность)
(c) L связана с V: высокий L ⟹ s ∈ V
```

**Категориальная формулировка:**
```
L — морфизм в категорию (ℝ≥0, ≤)
Диссипативность: L·F ≤ L (относительно порядка)
```

**Реализации:**

| Архитектура | L (функция стабильности) |
|-------------|--------------------------|
| SYNARC | VIT или W (функция диссипации) |
| FEP | −F (negative free energy) |
| Control | V из теории управления |
| Thermodynamics | −S (negative entropy) |

**Критерий выполнения:** Явная функция с доказательством свойств
**Критерий провала:** Стабильность "предполагается" без формализации

### Ω3.4 Аттракторы

> Существуют ли устойчивые состояния?

**Универсальное требование:**
```
∃ A ⊂ S (аттрактор):
∀s₀ ∈ Basin(A): lim_{t→∞} s(t) ∈ A
```

**Критерий выполнения:** Теорема существования аттрактора (или доказательство отсутствия)
**Критерий провала:** Отсутствие анализа асимптотического поведения

### Ω3.5 Термодинамическая согласованность

> Согласована ли динамика с физическими законами?

**Универсальное требование:**
```
Динамика удовлетворяет:
(a) Второму началу термодинамики (локально или глобально)
(b) Флуктуационно-диссипативной теореме (если стохастична)
(c) Границам Ландауэра (для вычислительных операций)
```

**Критерий выполнения:** Явное указание на соблюдение или обоснование нарушения
**Критерий провала:** Игнорирование термодинамики

## Ω4. КОГНИЦИЯ

*Категория: Обработка информации*

### Ω4.1 Предиктивная модель

> Содержит ли система модель для предсказания?

**Универсальное требование:**
```
∃ предиктивная модель M: S × H → P(E)
M(s, history) = распределение ожидаемых наблюдений
```

**Категориальная формулировка:**
```
M — функтор State × History → Prob(Env)
Естественность: изменение состояния согласовано с изменением предсказаний
```

**Реализации:**

| Архитектура | Форма предиктивной модели |
|-------------|---------------------------|
| SYNARC | p(e|s) через генеративную модель |
| FEP | Generative model q(e, s) |
| Predictive Coding | Иерархическое предсказание ошибок |
| Bayesian Brain | Posterior p(s|e) |

**Критерий выполнения:** Явная предиктивная модель с обновлением
**Критерий провала:** Чисто реактивная система без предсказания

### Ω4.2 Обновление по наблюдениям

> Как система интегрирует новую информацию?

**Универсальное требование:**
```
∃ оператор обновления U: S × O → S
s' = U(s, observation)
U согласован с предиктивной моделью (байесовски или аналогично)
```

**Критерий выполнения:** Явное правило обновления с обоснованием
**Критерий провала:** Обновление "через обучение" без спецификации

### Ω4.3 Информационная геометрия

> Определена ли информационная структура пространства состояний?

**Универсальное требование:**
```
∃ информационная метрика G: S → ℝ^{D×D}
G определяет "стоимость" перехода между состояниями
```

**Реализации:**

| Архитектура | Информационная метрика |
|-------------|------------------------|
| SYNARC | Метрика Фишера Γ из p(e|s) |
| FEP | Fisher information metric |
| Information Geometry | Любая α-connection |
| Neural | Pullback метрики из embedding |

**Критерий выполнения:** Метрика выведена из информационных соображений
**Критерий провала:** Евклидова метрика "по умолчанию"

### Ω4.4 Выбор действий

> Как система выбирает действия?

**Универсальное требование:**
```
∃ политика π: S → P(A) или π: S → A
π оптимизирует некоторый функционал (явный или неявный)
```

**Категориальная формулировка:**
```
π — морфизм State → Actions (или Prob(Actions))
Оптимальность: π = argmax/argmin некоторого функционала
```

**Критерий выполнения:** Явная политика с критерием оптимальности
**Критерий провала:** "Агент выбирает действие" без спецификации

## Ω5. ДЕЙСТВИЕ

*Категория: Влияние на среду*

### Ω5.1 Сенсомоторный интерфейс

> Как система связана со средой?

**Универсальное требование:**
```
Sensors: E → S_sensory (восприятие)
Actuators: S_motor → E (действие)
Замыкание: E ←→ S через perception-action loop
```

**Категориальная формулировка:**
```
Биекция: (Sensors, Actuators) ≅ Markov blanket системы
Сенсоры и актуаторы образуют пределы в категории взаимодействий
```

**Критерий выполнения:** Явный цикл восприятие-действие
**Критерий провала:** "Ввод/вывод" без физической спецификации

### Ω5.2 Модель тела

> Представлено ли тело (воплощение) в пространстве состояний?

**Универсальное требование:**
```
∃ S_body ⊂ S: модель физического воплощения
S_body включает: проприоцепцию, интероцепцию, схему тела
```

**Критерий выполнения:** Явное S_body с динамикой
**Критерий провала:** Disembodied intelligence (если система воплощена)

### Ω5.3 Affordances

> Как среда представлена в терминах возможностей действия?

**Универсальное требование:**
```
affordance(e, s) = {a ∈ Actions : действие a возможно и улучшает состояние}
Affordances определяются через функцию существования Φ или её аналог
```

**Критерий выполнения:** Affordances выводятся из внутренней структуры системы
**Критерий провала:** Отдельный "perception module" без связи с действием

## Ω6. РЕФЛЕКСИВНОСТЬ

*Категория: Самонаблюдение (Кибернетика 2-го порядка)*

### Ω6.1 Рефлексивный оператор

> Определён ли оператор самонаблюдения?

**Универсальное требование:**
```
∃ φ: S → S (рефлексивный оператор):
φ(s) = модель состояния s, построенная системой
```

**Категориальная формулировка:**
```
φ — эндофунктор в категории состояний
φ: State → State с естественным преобразованием η: Id → φ
```

**Реализации:**

| Архитектура | φ (рефлексивный оператор) |
|-------------|---------------------------|
| SYNARC | φ = A ∘ P ∘ O (Observe → Predict → Act) |
| FEP | Active inference self-model |
| IIT | Φ-structure self-reference |
| Metacognition | Explicit self-monitoring |

**Критерий выполнения:** Явная структура φ с семантикой
**Критерий провала:** "Система знает своё состояние" без формализации

### Ω6.2 Сходимость самомодели

> Сходится ли итерация φ?

**Универсальное требование:**
```
∃ механизм обеспечения сходимости:
(a) Контракция: d(φ(s₁), φ(s₂)) ≤ k·d(s₁, s₂), k < 1
(b) Или: φ^n(s) → s* (фиксированная точка)
```

**Категориальная формулировка:**
```
φ имеет terminal coalgebra: ∃ ν такой что φ(ν) ≅ ν
```

**Критерий выполнения:** Доказательство сходимости (контракция или аналог)
**Критерий провала:** Расходящаяся или осциллирующая самомодель

### Ω6.3 Когерентность

> Определена ли мера согласованности между s и φ(s)?

**Универсальное требование:**
```
∃ ρ: S → [0, 1] (мера когерентности):
ρ(s) = степень согласованности реального и моделируемого состояния
```

**Реализации:**

| Архитектура | ρ (мера когерентности) |
|-------------|------------------------|
| SYNARC | ρ_RC = 1 - d_Γ(s, φ(s))/d_max |
| FEP | 1 - prediction_error |
| Metacognition | calibration score |
| IIT | Φ value |

**Критерий выполнения:** Количественная мера с порогами
**Критерий провала:** Отсутствие меры самопонимания

### Ω6.4 Уровни метакогниции (Кибернетика порядка N)

> Определены ли уровни самореференции?

**Универсальное требование:**
```
Level 0: модель мира
Level 1: модель себя как моделирующего мир (порядок 2)
Level 2: модель себя как моделирующего себя (порядок 3)
...
Level N: φ^N(s) ≈ φ^{N-1}(s) (сходимость на уровне N)
```

**Критерий выполнения:** Явное N с доказательством конечности
**Критерий провала:** Бесконечная регрессия без сходимости

### Ω6.5 Интроспективная полнота

> Какая часть состояния доступна рефлексии?

**Универсальное требование:**
```
r = dim(Im(O)) / dim(S) ∈ (0, 1)
где O — оператор самонаблюдения
```

**Критерий выполнения:** Явное r с обоснованием (частичная интроспекция)
**Критерий провала:** r = 1 (невозможная полная интроспекция) или r = 0

## Ω7. АБСТРАКЦИЯ

*Категория: Генерализация и символизация*

### Ω7.1 Механизм генерализации

> Как знание об одном переносится на другое?

**Универсальное требование:**
```
∃ отношение эквивалентности ~ или симилярности:
s_A ~ s_B ⟹ π(s_A) ≈ π(s_B)
где π — проекция на релевантное подпространство
```

**Категориальная формулировка:**
```
Генерализация — функтор из конкретной категории в абстрактную
G: Concrete → Abstract, counit: G·I → Id (instantiation)
```

**Критерий выполнения:** Явное ~ с обоснованием (эмерджентное, не supervised)
**Критерий провала:** Генерализация только "через обучение на примерах"

### Ω7.2 Формирование категорий

> Как возникают классы объектов?

**Универсальное требование:**
```
Category(w) = {s ∈ S : s сходится к аттрактору w}
Иерархия: Category(w₁) ⊂ Category(w₂) ⟺ Basin(w₁) ⊂ Basin(w₂)
```

**Критерий выполнения:** Категории как эмерджентные структуры динамики
**Критерий провала:** Категории как заданные классы или кластеры

### Ω7.3 Символическая система

> Определены ли символы и их семантика?

**Универсальное требование:**
```
Symbol w ⟺:
(a) w — стабильный паттерн (аттрактор, концепт, ...)
(b) w имеет basin of attraction (множество референтов)
(c) w имеет grounding (связь с сенсомоторикой или другими символами)
```

**Категориальная формулировка:**
```
Символы — объекты в свободно-порождённой категории
Семантика — функтор из символов в денотаты
```

**Критерий выполнения:** Формальное определение символов с (a-c)
**Критерий провала:** "Токен", "эмбеддинг" без динамической семантики

### Ω7.4 Композициональность

> Как комбинируются символы?

**Универсальное требование:**
```
∃ оператор композиции ⊗: S × S → S
(a) ⊗ сохраняет жизнеспособность: w₁ ⊗ w₂ ∈ V
(b) ⊗ непрерывна (или структурно согласована)
(c) Частичная обратимость: ∃ операции декомпозиции
```

**Критерий выполнения:** Алгебраическая структура с (a-c)
**Критерий провала:** Конкатенация или attention без алгебры

### Ω7.5 Абстрактные символы

> Как возникают символы без прямого референта?

**Универсальное требование:**
```
Grounding levels:
Level 1: прямое сенсомоторное заземление
Level 2: производное заземление (через связи с Level 1)
Level 3: чисто реляционное (определён только через отношения)

∃ механизм порождения Level 3 из Level 1-2
```

**Критерий выполнения:** Явный механизм абстрагирования
**Критерий провала:** Абстрактные символы как отдельный "модуль"

## Ω8. ТЕМПОРАЛЬНОСТЬ

*Категория: Время и память*

### Ω8.1 Представление времени

> Как система представляет время?

**Универсальное требование:**
```
Хронос: t ∈ ℝ или ℕ (равномерное время)
Кайрос: T_K = {t_i : событие качественного изменения}
Субъективное время может отличаться от физического
```

**Критерий выполнения:** Различение объективного и субъективного времени
**Критерий провала:** Только дискретные timesteps без семантики

### Ω8.2 Память и консолидация

> Определены ли механизмы памяти?

**Универсальное требование:**
```
∃ память M: (S, T) → Storage с операциями:
(a) write(m, salience): запись с весом важности
(b) read(query) → m: извлечение по запросу
(c) consolidate(): укрепление важного, забывание неважного
```

**Категориальная формулировка:**
```
Память — comonadic структура: M: State → F(State)
extract: F(State) → State (текущее состояние)
extend: F(State) → F(F(State)) (история историй)
```

**Критерий выполнения:** Явная структура памяти с динамикой
**Критерий провала:** "Веса нейросети" как единственная память

### Ω8.3 Планирование

> Как система предвидит будущее?

**Универсальное требование:**
```
∃ механизм симуляции будущих траекторий:
Plan(s, [a₁, ..., a_n]) = [s, F(s,a₁), F(F(s,a₁),a₂), ...]
Оценка: Value(plan) через Φ или аналог
```

**Критерий выполнения:** Явный механизм симуляции с оценкой
**Критерий провала:** "Planning module" без связи с динамикой

### Ω8.4 Иерархия горизонтов

> Определены ли разные временные масштабы?

**Универсальное требование:**
```
Φ_τ(s) = ∫₀^∞ e^{-t/τ} · φ(s(t)) dt (для непрерывного)
или Φ_τ = Σ_{i=0}^H γ^i · φ(s_{t+i}) (для дискретного)

Multi-horizon: Φ_combined = Σᵢ αᵢ · Φ_{τᵢ}
```

**Критерий выполнения:** Multi-horizon планирование с адаптивными весами
**Критерий провала:** Фиксированный горизонт

## Ω9. СОЦИАЛЬНОСТЬ

*Категория: Интерсубъективность и композиция*

### Ω9.1 Модель других агентов

> Как система моделирует других?

**Универсальное требование:**
```
∀ agent i: ∃ модель φᵢ(s, e) в S
Theory of Mind: предсказание состояний и действий других через φᵢ
```

**Критерий выполнения:** Явные модели других агентов
**Критерий провала:** Другие агенты как часть "среды" без внутренних состояний

### Ω9.2 Мульти-агентная динамика

> Определена ли динамика взаимодействия?

**Универсальное требование:**
```
s_joint = (s₁, ..., s_n) ∈ S₁ × ... × S_n
F_joint определяет совместную эволюцию
Равновесие: Nash или cooperative или иное с обоснованием
```

**Категориальная формулировка:**
```
Продуктная категория: Agent₁ × ... × Agent_n
Взаимодействие: естественные преобразования между агентами
```

**Критерий выполнения:** Теоретико-игровая или иная формализация
**Критерий провала:** "Multi-agent" без формальной динамики

### Ω9.3 Коммуникация

> Как передаётся информация между агентами?

**Универсальное требование:**
```
Communication channel: S_i → Symbols → S_j
Свойства: honesty measure, grounding overlap, bandwidth
```

**Критерий выполнения:** Формальный протокол коммуникации с гарантиями
**Критерий провала:** "Emergent communication" без спецификации

### Ω9.4 Фрактальное замыкание

> Образуют ли агенты мета-агента той же структуры?

**Универсальное требование:**
```
A_∪ = compose(A₁, ..., A_n)
A_∪ удовлетворяет тем же аксиомам/условиям, что Aᵢ
```

**Категориальная формулировка:**
```
Композиция — функтор ∪: Agent^n → Agent
Фрактальность: ∪ сохраняет все существенные структуры
```

**Реализации:**

| Архитектура | Фрактальное замыкание |
|-------------|----------------------|
| SYNARC | φ_∪, ρ_RC_∪, σ_sys_∪ — Theorem 13.2 |
| FEP | Nested Markov blankets |
| Autopoiesis | Организации организаций |
| Complex Systems | Self-similar hierarchies |

**Критерий выполнения:** Теорема о замыкании под композицией
**Критерий провала:** Мета-агент требует новых принципов

## Ω10. РЕАЛИЗУЕМОСТЬ

*Категория: Вычислительные и физические ограничения*

### Ω10.1 Вычислительная сложность

> Какова сложность операций системы?

**Универсальное требование:**
```
T(n) = O(f(D)) где D — размерность или размер
f — полином (не экспонента) для основных операций
```

**Критерий выполнения:** Полиномиальная сложность с доказательством
**Критерий провала:** Экспоненциальная или неизвестная сложность

### Ω10.2 Аппроксимации

> Какие аппроксимации используются?

**Универсальное требование:**
```
∀ approximation A:
(a) Исходная точная формула
(b) Аппроксимация
(c) Погрешность ε(A) и условия её малости
(d) Обоснование выбора
```

**Критерий выполнения:** Все аппроксимации задокументированы с ошибками
**Критерий провала:** "Можно аппроксимировать" без анализа

### Ω10.3 Ресурсные ограничения

> Определены ли бюджеты ресурсов?

**Универсальное требование:**
```
compute ≤ C_max (вычисления в единицу времени)
memory ≤ M_max (хранение состояния)
energy ≤ E_max (энергопотребление)
latency ≤ L_max (время реакции)
```

**Критерий выполнения:** Явные ограничения с обоснованием
**Критерий провала:** "Достаточно ресурсов" без спецификации

### Ω10.4 Масштабируемость

> Как система масштабируется?

**Универсальное требование:**
```
Performance(α·D) ≥ β · Performance(D)
для некоторых α > 1, β > 0 (sublinear degradation)
```

**Критерий выполнения:** Анализ scaling laws
**Критерий провала:** "Работает на данном размере"

### Ω10.5 Физическая реализуемость

> Совместима ли архитектура с физикой?

**Универсальное требование:**
```
Архитектура не нарушает:
(a) Термодинамику (Landauer bound)
(b) Теорию информации (channel capacity)
(c) Причинность (скорость света)
(d) Квантовую механику (если применимо)
```

**Критерий выполнения:** Явная проверка физических ограничений
**Критерий провала:** Implicit assumptions о неограниченных ресурсах

## Ω11. ТЕЛЕОЛОГИЯ

### Ω11.1 Происхождение целей

> Откуда возникают цели?

**Универсальное требование:**
```
Goals выводятся из функции существования Φ (или аналога)
Goals = {s* : Φ(s*) — локальный оптимум}
или Goals определяются через декомпозицию Φ
```

**Критерий выполнения:** Цели эмерджентны из внутренней структуры
**Критерий провала:** Reward function задана извне без обоснования

### Ω11.2 Иерархия целей

> Как организованы цели?

**Универсальное требование:**
```
Goals = {Survival ⊃ Homeostatic ⊃ Instrumental ⊃ Epistemic ⊃ ...}
Частичный порядок с формальным отношением
```

**Критерий выполнения:** Явная иерархия с доказательством структуры
**Критерий провала:** Плоский список целей или rewards

### Ω11.3 Разрешение конфликтов

> Как разрешаются конфликты целей?

**Универсальное требование:**
```
∃ единый механизм M:
M разрешает conflict(g₁, g₂) через:
(a) Единый скаляр Φ, или
(b) Nash-равновесие подсистем, или
(c) Иерархическое доминирование
```

**Критерий выполнения:** Единый принцип разрешения с обоснованием
**Критерий провала:** Ad-hoc правила приоритетов

```
affect(s) = f(Φ(s), Φ̇(s), ∇Φ(s), ...)
```

```
affect модулирует параметры системы:

```

## Ω13. ВЕРИФИКАЦИЯ

*Категория: Фальсифицируемость и тестирование*

### Ω13.1 Эмпирическая фальсифицируемость

> Какие предсказания можно опровергнуть?

**Универсальное требование:**
```
∃ prediction P:
¬P ⟹ архитектура ошибочна
P формулируется до эксперимента (не post-hoc)
```

**Критерий выполнения:** Минимум 3 фальсифицируемых предсказания
**Критерий провала:** Все предсказания post-hoc или нефальсифицируемы

### Ω13.2 Минимальный тест

> Какой простейший эксперимент проверит принципы?

**Универсальное требование:**
```
Test = (setup, metric, success_threshold, failure_threshold)
Время: O(days-weeks), не O(years)
```

**Критерий выполнения:** Конкретный протокол с метриками
**Критерий провала:** "Создать полную реализацию и посмотреть"

### Ω13.3 Режимы отказа

> Какие failure modes предсказаны?

**Универсальное требование:**
```
∀ condition C: определён режим отказа при нарушении C
Анализ: graceful degradation vs catastrophic failure
```

**Критерий выполнения:** Анализ каждого условия на режимы отказа
**Критерий провала:** "Система безопасна" без анализа

### Ω13.4 Сравнение с альтернативами

> Как архитектура соотносится с конкурентами?

**Универсальное требование:**
```
∀ competitor C: ∃ dimension D:
This ≷ C on D с обоснованием и признанием trade-offs
```

**Критерий выполнения:** Честное сравнение с trade-offs
**Критерий провала:** "Наша архитектура лучшая"

- Все условия Ω1-Ω5 (базовая когниция)
- Условия Ω6.1-Ω6.N (N уровней рефлексии)
- Ω9.4 с рекурсивным применением (системы порядка N образуют систему порядка N)
